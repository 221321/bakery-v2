# ВкусноЦех 2.0 — производство, доставка, 1С

Объединение опыта двух проектов: каркас и 1С-интеграция из ЖАЙЫК АКТАУ + производственная логика из ВкусноЦех.
Чистый Node.js без внешних зависимостей. Хранилище: JSON-файлы в `./data` (позже — PostgreSQL).

## Роли и цепочка

1. **Завхоз** — закуп: справочник сырья, документы «Поступление» (цены обновляются документом), согласование заявок пекарей на новое сырьё. Поступления уходят в 1С («Поступление ТМЗ» + «Установка цен номенклатуры»).
2. **Пекарь** — только рецепты и граммажи (цен не видит), план на день из заказов, журнал выпуска за смену. Выпуск увеличивает остатки и уходит в 1С («Отчёт производства за смену»).
3. **Менеджер** — заносит заказы клиентов с вечера, устанавливает цены продажи (видит себестоимость и маржу), клиенты, отчёты, долги, сотрудники.
4. **Экспедитор** — очередь на день, доставка, оплата нал/QR/долг/частично, погашение долгов. Доставленный заказ уходит в 1С («Реализация ТМЗ и услуг»).

## Запуск

```
node seed.js     # первичная инициализация (идемпотентно, данные не перезаписывает)
node server.js   # порт 3000 (или PORT из окружения)
```

Логины по умолчанию: manager/manager123, zavhoz/zavhoz123, baker1/baker123, exp1/exp123 — сменить после первого входа.

## Деплой на VPS (bakery.probuh.asia)

```
cd /var/www && git clone <repo> bakery-v2 && cd bakery-v2
node seed.js
pm2 start server.js --name bakery-v2
pm2 save
```
nginx: скопировать конфиг zhaiyk, заменить домен на bakery.probuh.asia и порт на 3000 (или свободный), затем `certbot --nginx -d bakery.probuh.asia`.

Обновление: правка на GitHub → `git pull` → `pm2 restart bakery-v2`.

## API для расширения 1С (заголовок X-Sync-Key)

Ключ: переменная окружения SYNC_KEY (по умолчанию `bakery_1c_2026` — сменить в проде!).

- `GET /api/1c/receipts` — новые поступления → Поступление ТМЗ + Установка цен; `POST /api/1c/receipts/:id/mark-exported`
- `GET /api/1c/production` — выпуски за смену → Отчёт производства за смену; `POST /api/1c/production/:id/mark-exported`
- `GET /api/1c/orders` — доставленные, не проведённые → Реализация ТМЗ и услуг; `POST /api/1c/orders/:id/mark-realized`
- `GET /api/1c/debt-settlements` — погашения → ПКО; `POST /api/1c/debt-settlements/:id/mark-exported`
- `POST /api/1c/clients/sync` — загрузка контрагентов из 1С (name, code, phone, address)
- `POST /api/1c/products/sync` — коды номенклатуры готовой продукции из 1С

Расширение 1С делается копией ЖайыкАктауAPI: модуль синхронизации тот же, меняются URL и добавляются две загрузки (поступления и производство). Все известные грабли актуальны: табличная часть «Товары», план счетов «Типовой», валюта 398/курс 1, заполнение счетов учёта через серверные обработчики.
